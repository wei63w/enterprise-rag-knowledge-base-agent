## 2026-05-21：Phase 3 向量索引与基础检索问答

阶段：Phase 3

状态：已完成

提交记录：`feat: add phase 3 vector index and chat`

### 新增内容

- 新增 Embedding 服务接口 `EmbeddingService` 和 DeepSeek 实现 `DeepSeekEmbeddingService`。
- 新增 Embedding 配置类 `EmbeddingProperties`。
- 新增 Milvus 向量服务 `MilvusService`，包括 Collection 初始化、向量写入和向量检索。
- 新增 Milvus 配置类 `MilvusProperties`。
- 新增 Chat 服务 `ChatService`，编排 RAG 流程：向量检索 → prompt 构建 → LLM 调用。
- 新增问答历史实体 `ChatHistoryEntity` 和仓储 `ChatHistoryRepository`。
- 新增 Chat API 控制器 `ChatController`，提供 `/api/chat` POST 接口。
- 新增 Chat 请求 DTO `ChatRequest`、响应 DTO `ChatResponse`、来源引用 DTO `SourceReference`。
- 新增前端 Chat API 服务 `chatApi.ts`。
- 新增 Phase 3 实施计划文档和设计文档。

### 修改内容

- 更新后端依赖，加入 LangChain4j 0.35.0、LangChain4j-OpenAI 0.35.0、LangChain4j-Milvus 0.35.0。
- 更新后端配置，加入 embedding、chat、deepseek 配置项。
- 更新 `RagApplication.java`，启用 `EmbeddingProperties` 和 `MilvusProperties` 配置扫描。
- 更新 `DocumentService.java`，添加 EmbeddingService 和 MilvusService 依赖，文档上传后自动触发向量化。
- 更新 `DocumentServiceTest.java`，添加 EmbeddingService 和 MilvusService mock。
- 更新前端 `App.tsx`，Chat 页接入真实问答，展示答案和来源引用。
- 更新前端 `styles.css`，添加 Chat 输入区、来源引用和占位提示样式。

### 验证结果

- `mvn -f backend/pom.xml test` 通过，共 17 个后端测试。
- `npm --prefix frontend test` 通过，共 2 个前端测试。
- `npm --prefix frontend run build` 通过。
- `npm --prefix frontend audit --audit-level=high` 通过，未发现高危漏洞。
- `docker compose --env-file .env.example -f docker/docker-compose.yml config --quiet` 通过。

### 后续影响

- Phase 4 可以直接在 Chat API 响应中返回更详细的引用来源（文档名、页码等）。
- 前端可以展示引用片段详情和相关性分数。
- 文档上传后会自动生成向量索引，无需手动触发。

## 2026-05-21：Phase 0 仓库与文档初始化

阶段：项目管理文档补充

状态：已完成

提交记录：`feat: add document processing flow`

### 新增内容

- 新增文档状态枚举 `DocumentStatus`。
- 新增文档实体 `DocumentEntity` 和切片实体 `ChunkEntity`。
- 新增 JPA 仓储 `DocumentRepository` 和 `ChunkRepository`。
- 新增固定长度切片器 `FixedLengthChunker`。
- 新增 Apache Tika 文本解析器 `DocumentTextParser`。
- 新增原始文件存储接口 `StorageService` 和 MinIO 实现 `MinioStorageService`。
- 新增文档处理服务 `DocumentService`，负责上传、存储、解析、切片和状态流转。
- 新增文档 API：上传、列表、详情、状态查询和删除。
- 新增统一错误响应和非法参数 400 处理。
- 新增前端文档 API 封装 `documentApi.ts`。
- 前端文档页新增真实列表加载、上传、删除、加载状态和错误提示。
- 新增 Phase 2 实施计划文档。

### 修改内容

- 更新后端依赖，加入 Spring Data JPA、Validation、Apache Tika、MinIO、MySQL、H2 和 `commons-io`。
- 更新后端配置，加入 MySQL 数据源和 JPA 自动建表配置。
- 更新 Vite 配置，增加 `/api` 到后端的开发代理。
- 更新 `.env.example`，加入 MySQL 端口和 Spring 数据源变量。
- 更新 Docker Compose，让 MySQL 映射端口可通过 `MYSQL_PORT` 配置。
- 更新 README，说明本机 `3306` 端口冲突时如何切换到 `3307`。
- 更新前端移动端样式，避免窄屏下标题和表格被压成竖排。

### 验证结果

- `mvn -f backend/pom.xml test` 通过，共 9 个后端测试。
- `npm --prefix frontend test` 通过，共 2 个前端测试。
- `npm --prefix frontend run build` 通过。
- `npm --prefix frontend audit --audit-level=high` 通过，未发现高危漏洞。
- `docker compose --env-file .env.example -f docker/docker-compose.yml config --quiet` 通过。
- 使用 MySQL `3307` 和 MinIO 完成真实 TXT 上传验证，返回状态 `READY`，切片数为 `1`。
- 浏览器验证前端显示真实上传文档、`READY` 状态和上传入口。

### 后续影响

- Phase 3 可以直接基于 `chunks` 表内容生成 embedding 并写入 Milvus。
- 文档删除当前会同步删除 MySQL 记录、切片记录和 MinIO 原始文件。
- 本机如果已有 MySQL 占用 `3306`，可以将 `.env` 中 `MYSQL_PORT` 和 `SPRING_DATASOURCE_URL` 切换到 `3307`。
