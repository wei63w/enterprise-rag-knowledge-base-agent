## Phase 3：向量索引与基础检索问答

状态：已完成
完成日期：2026-05-21

### 目标

接入 embedding 和 Milvus，让系统可以基于已处理文档执行基础向量召回，并通过 LLM 生成回答。

### 已完成任务

- 实现 Embedding 服务接口 `EmbeddingService` 和 DeepSeek 实现 `DeepSeekEmbeddingService`。
- 建立 Milvus 向量服务 `MilvusService`，包括 Collection 初始化、向量写入和向量检索。
- 将切片向量写入 Milvus（文档上传后自动触发向量化）。
- 实现 Chat 服务 `ChatService`，编排 RAG 流程：向量检索 → prompt 构建 → LLM 调用。
- 实现 Chat API `/api/chat`，支持提交问题并返回答案。
- 保存问答历史到 MySQL（`ChatHistoryEntity`）。
- 前端对话页接入真实问答请求，展示答案和来源引用。

## Phase 6b：BM25 与 Rerank 检索增强

状态：已完成
完成日期：2026-05-22
提交记录：`feat: integrate BM25 search and RRF merge into ChatService`

### 目标

增加 BM25 关键词召回和 RRF 融合排序，提升检索质量。

### 已完成任务

- 新增 RagProperties 中 bm25TopK、rrfTopK 配置项。
- 新增 BM25KeywordService，使用 Apache Lucene 内存索引对切片建关键词索引。
- DocumentService 上传时同步写入 BM25 索引，删除时同步清理。
- ChatService 检索流程改为双路检索：向量 topK×2 + BM25 topK×2。
- ChatService 新增 RRF（Reciprocal Rank Fusion）融合排序，取最终 topK。
