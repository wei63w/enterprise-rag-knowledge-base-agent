## Phase 2：文档处理主链路

状态：已完成

完成日期：2026-05-21

提交记录：`feat: add document processing flow`

### 目标

实现 MVP 的文档处理入口，让用户可以上传文档，系统可以保存原始文件、解析文本、生成切片并记录处理状态。

### 已完成任务

- 实现文档实体、切片实体和 JPA 数据访问层。
- 后端通过 JPA 自动建表，连接 MySQL 保存文档和切片记录。
- 实现文档上传接口，支持 PDF、Markdown、TXT。
- 实现 MinIO 原始文件存储。
- 使用 Apache Tika 提取文本，并补充解析器测试。
- 实现固定长度切片策略。
- 将文档状态从 `PROCESSING` 更新为 `READY` 或 `ERROR`。
- 实现文档列表、详情、状态和删除接口。
- 前端文档管理页接入上传、列表、状态展示和删除入口。
- 修复移动端文档页布局压缩问题。

### 验证结果

- `mvn -f backend/pom.xml test` 通过。
- `npm --prefix frontend test` 通过。
- `npm --prefix frontend run build` 通过。
- `npm --prefix frontend audit --audit-level=high` 通过。
- `docker compose --env-file .env.example -f docker/docker-compose.yml config --quiet` 通过。
- 使用 MySQL `3307` 和 MinIO 完成真实 TXT 上传验证，返回状态 `READY`，切片数为 `1`。
- 浏览器验证前端可以显示真实上传文档、`READY` 状态和上传入口。

## Phase 3：向量索引与基础检索问答

状态：未开始

### 目标

接入 embedding 和 Milvus，让系统可以基于已处理文档执行基础向量召回，并通过 LLM 生成回答。

### 待办任务

- 实现 embedding 服务接口和默认模型配置。
- 建立 Milvus collection 初始化逻辑。
- 将切片向量写入 Milvus。
- 实现向量检索服务。
- 实现 Chat API，支持提交问题并返回答案。
- 保存问答历史和模型使用信息。
- 前端对话页接入真实问答请求。

### 完成标准

- 已处理文档可以生成向量索引。
- 用户提问后可以得到基于知识库的回答。
- 无相关内容时返回“不具备足够依据”的兜底回答。
- 问答记录可持久化。

## Phase 4：答案溯源与前端联动

状态：未开始

### 目标

让每个知识库回答都能展示引用来源，用户可以查看答案依据片段。

### 待办任务

- 定义引用来源 DTO。
- 在 Chat API 响应中返回来源列表。
- 实现来源片段查询接口。
- 前端展示文档名、片段文本、相关性分数和页码。
- 支持展开或点击查看引用详情。

### 完成标准

- 每个基于知识库生成的答案至少展示一个引用来源。
- 引用片段内容可在前端查看。
- 缺失页码时不影响展示。

## Phase 5：异常处理与一致性补强

状态：未开始

### 目标

补齐 MVP 中关键失败场景的用户提示、状态记录和数据一致性处理。

### 待办任务

- 实现统一错误响应结构。
- 补齐上传、解析、切片、向量化、检索、LLM 调用失败处理。
- 实现文档删除的一致性流程。
- 增加失败状态和错误信息展示。
- 补齐异常场景测试。

### 完成标准

- 常见失败场景不会导致文档永久停留在处理中。
- 删除失败时能标记并提示可重试。
- 前端展示用户可理解的错误信息。
